PROMPT 3: Program Working Explanation
----------------------------------------------------------------
Can you explain how the program works internally?
How does each drawing algorithm work?

RESPONSE 3:
----------------------------------------------------------------
Claude explained the working of the program:

1. CANVAS STORAGE:
   - A 2D array char picture[24][80] stores the entire drawing
   - Every cell is initialized to '_' (empty/background)
   - When a shape is drawn, cells are changed to '*'

2. BRESENHAM'S LINE ALGORITHM (drawLine):
   - Efficiently draws a line between two points (x1,y1)-(x2,y2)
   - Uses integer arithmetic to decide which pixel to plot next
   - Calculates error term to stay as close to true line as possible
   - Works for lines in all directions and slopes

3. MIDPOINT CIRCLE ALGORITHM (drawCircle):
   - Takes center (cx, cy) and radius as input
   - Uses 8-way symmetry to draw all 8 octants simultaneously
   - Only calculates pixels for one octant and mirrors to others
   - Decision parameter 'd' determines next pixel position

4. RECTANGLE (drawRectangle):
   - Draws 4 lines: top, bottom, left, right edges
   - Uses drawLine() for each of the four sides
   - Creates a hollow rectangle outline

5. TRIANGLE (drawTriangle):
   - Connects three given vertices with three lines
   - Calls drawLine() three times for each edge

6. DELETE OPERATION:
   - Object is marked as inactive (active = 0)
   - redrawAll() is called which clears canvas
   - All remaining active objects are redrawn from scratch
   - This cleanly removes the deleted shape

7. OBJECT STORAGE:
   - Each object stored in a struct with type and coordinates
   - Type: 1=Line, 2=Rectangle, 3=Circle, 4=Triangle
   - Maximum 50 objects can be stored simultaneously
